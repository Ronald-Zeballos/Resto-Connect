# Sprint 2 - Reportes y operacion

## Objetivo

Completar el frente operativo y analitico del restaurante.

## Alcance propuesto

- Modulo `reportes` por casos de uso:
  - ventas por rango
  - productos mas vendidos
  - ventas por metodo de pago
  - ventas por mesero
  - consumo inventario por periodo
  - comparacion consumo real vs predicho
- Modulo `incidencias` con registro, seguimiento y notificaciones.
- Modulo `personal` para gestion operativa basica.
- Consultas mas afinadas para panel de cocina y panel de mesero.
- Tests adicionales sobre reportes y escenarios cruzados.

## Criterio de cierre

- Endpoints analiticos funcionales.
- Casos de uso trazables y sin servicios monoliticos.
- Respuestas REST listas para frontend administrativo.
